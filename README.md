## Featured Projects

<details open>
<summary><b>MacDeck — Real-Time Mac System & Media Companion</b></summary>

<br/>

> A landscape-first, installable system and media dashboard that transforms a secondary screen, phone, tablet, or browser into a real-time visual companion for a Mac.

| Dimension | Engineering Detail |
| :--- | :--- |
| **Stack** | Python, Flask, psutil, JavaScript, HTML5, CSS3, PWA, Service Workers |
| **Architecture** | Dual-mode provider architecture with Live API and deterministic Demo Provider |
| **Telemetry** | CPU, RAM, network I/O, storage, battery percentage, charging state |
| **Media** | Metadata, album artwork, progress, previous/play-pause/next controls |
| **Testing** | unittest, pytest, Node.js VM-based tests, Playwright |
| **Browsers** | Chromium and WebKit acceptance coverage |
| **Deployment** | GitHub Actions, GitHub Pages, custom Python static exporter |
| **Live Product** | [Launch MacDeck](https://raghavvxts.github.io/macdeck/) |

MacDeck uses a **Python Flask backend with `psutil`** for live Mac telemetry and HTTP JSON endpoints including `/api/stats`. A provider abstraction allows the same vanilla JavaScript frontend to switch between actual system data and a deterministic public demo runtime.

The product includes responsive landscape density tiers, low-battery and critical-battery states, custom PWA installation UX, Add to Home Screen behavior, deployment-aware service-worker caching, offline capability, and layouts specifically hardened for short-landscape mobile displays.

**Engineering highlights:** no React, no TypeScript, no Electron, no Tauri, no WebSocket dependency, and no native macOS shell. The architecture stays intentionally lightweight and browser-first.

</details>
